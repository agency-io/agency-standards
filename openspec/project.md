# Project Context

## Purpose
`agency-standards` is an AI-powered architecture governance CLI for software projects. It inspects a project's structure, generates pytest-based architecture tests that enforce coding standards, and maintains a `CLAUDE.md` file for AI assistants. Standards are defined in YAML, test generation is delegated to Claude AI (via the Anthropic API), and custom code in generated files is preserved across regenerations.

## Tech Stack
- **Language:** Python 3.11+
- **CLI framework:** Typer (>=0.12.0)
- **Terminal output:** Rich (>=13.0.0)
- **Interactive prompts:** Questionary (>=2.0.0)
- **AI generation:** Anthropic Python SDK (>=0.40.0)
- **Config/standards format:** PyYAML (>=6.0.0)
- **Build system:** Hatchling via `pyproject.toml`
- **Package manager:** uv (distributed as a uv tool)
- **Linter/formatter:** Ruff (>=0.3.0)
- **Test runner:** pytest (>=8.0.0)
- **CI/CD:** GitHub Actions → PyPI via OIDC trusted publishing

## Project Conventions

### Code Style
- Line length: **100 characters** (enforced by ruff)
- Ruff rule sets: `E, F, I, W, UP, C90` (PEP 8, PyFlakes, isort, warnings, PyUpgrade, McCabe complexity)
- File names: `lowercase_with_underscores.py`
- Standard IDs and YAML filenames: `kebab-case` (e.g., `no-bare-except.yaml`)

### Architecture Patterns
- **Commands** live in `src/agency_standards/commands/` as individual modules (`init_cmd.py`, `update_cmd.py`, etc.)
- **Data models** are Python dataclasses defined in `models.py` (`Standard`, `ProjectContext`, `GeneratedFile`)
- **Separation of concerns:**
  - `inspector.py` — project introspection (language/type detection, file discovery)
  - `generator.py` — test file generation and custom-section preservation
  - `config.py` — read/write `.agency-standards.yaml`
  - `standards/loader.py` — load builtin and user-defined YAML standards
- **Standards** are YAML files with fields: `id`, `name`, `description`, `languages`, `tags`, `output_file`, `prompt`, `claude_md_section`
- **Generated test files** include a header marker (`# GENERATED by agency-standards | standard: <id>`) and a custom section marker (`# --- CUSTOM (preserved on update) ---`); everything after the custom marker is preserved on regeneration

### Testing Strategy
- The project does not yet have a unit/integration test suite for its own code
- Generated architecture tests (output of this tool) follow pytest conventions and live in `tests/architecture/` of target projects
- BDD tests are detected by presence of `.feature` files and step directories (`given/`, `when/`, `then/`)

### Git Workflow
- Single `main` branch; feature work via PRs
- Commit style: conventional commits (`feat:`, `fix:`, `docs:`, `chore:`)
- Releases trigger the PyPI publish GitHub Actions workflow

## Domain Context
- **Standards** are the core concept: each standard has an ID, targets one or more languages, and carries a Claude prompt used to generate an architecture test
- **Built-in standards** ship with the package under `src/agency_standards/builtin_standards/` (21 YAML files); users can add custom standards to `~/.agency-standards/standards/`
- **Project types** detected: `api`, `cli`, `library`, `unknown` (inferred from dependencies in `pyproject.toml`)
- **Languages** detected: Python, Go, TypeScript, Java
- **Claude Code skills** are installed to `.claude/commands/agency-standards/` in the target project, enabling AI-driven generation and updates from within Claude Code

## Important Constraints
- Python >=3.11 required (uses modern syntax and type hints)
- Anthropic API key must be present in the environment for test generation
- Generated file custom sections must never be overwritten — the `CUSTOM_MARKER` boundary is contractual
- Standard IDs must be unique across builtin and user-defined standards

## External Dependencies
- **Anthropic API** — used at generation time to produce pytest architecture tests from standard prompts
- **PyPI** — distribution target; published via GitHub Actions on release using OIDC trusted publishing
- **uv** — recommended install method (`uv tool install agency-standards`)
